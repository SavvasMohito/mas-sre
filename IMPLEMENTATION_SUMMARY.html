1 Implementation Summary

1.1 Multi-Agent Security Requirements Generation System

Date: October 2025 Framework: CrewAI Flows Status: Implementation Complete ✅

1.2 System Overview

This implementation delivers a fully functional multi-agent system that transforms high-level product requirements into comprehensive, standards-aligned security requirements using CrewAI’s Flow orchestration.

1.2.1 Architecture

The system consists of 5 specialized AI agents working in sequence with a self-evaluation loop:

Input (Text File)
    ↓
Requirements Analysis Agent
    ↓
┌──────────────────────────────────────┐
│  Domain Security Agent               │
│  LLM Security Specialist             │
│  Compliance Agent                    │
└──────────────────────────────────────┘
    ↓
Validation Agent
    ↓
[Self-Evaluation Decision]
    ↓
Output (JSON + Markdown)

1.3 Components Implemented

1.3.1 1. Core Agents (5 Crews)

1.3.1.1 Requirements Analysis Crew

  • Location: src/security_requirements_system/crews/requirements_analysis_crew/
  • Purpose: Parse high-level inputs and identify security implications
  • Output: Structured analysis of business features, data types, user roles, integrations, and security concerns

1.3.1.2 Domain Security Crew

  • Location: src/security_requirements_system/crews/domain_security_crew/
  • Purpose: Map requirements to OWASP, NIST, ISO 27001 controls
  • Tools: Weaviate vector database query tool
  • Output: Mapped security controls with implementation guidance

1.3.1.3 LLM Security Specialist Crew

  • Location: src/security_requirements_system/crews/llm_security_crew/
  • Purpose: Identify AI/ML components and add specialized controls
  • Threats Covered: Prompt injection, data leakage, model poisoning, adversarial attacks
  • Output: AI-specific security requirements when applicable

1.3.1.4 Compliance Crew

  • Location: src/security_requirements_system/crews/compliance_crew/
  • Purpose: Identify regulatory requirements (GDPR, HIPAA, PCI-DSS, etc.)
  • Output: Compliance controls, data handling rules, audit requirements

1.3.1.5 Validation Crew

  • Location: src/security_requirements_system/crews/validation_crew/
  • Purpose: Self-evaluation on 5 dimensions (completeness, consistency, correctness, implementability, alignment)
  • Output: Validation score (0-1), pass/fail, gaps, conflicts, recommendations

1.3.2 2. Flow Orchestration

Main Flow: src/security_requirements_system/main.py

  • Implements SecurityRequirementsFlow class extending CrewAI Flow
  • Self-evaluation loop with max 3 iterations
  • Validation threshold: 0.8
  • Automatic feedback integration on failed validation
  • Generates both JSON and Markdown outputs

1.3.3 3. Vector Database Integration

Weaviate Tools: - weaviate_tool.py: Custom CrewAI tool for querying security standards - weaviate_setup.py: Schema initialization and data ingestion utilities

Features: - Semantic search over security controls - Filter by standard (OWASP/NIST/ISO27001) - OpenAI text-embedding-3-small for vectorization - Supports local Weaviate (Docker) or cloud deployment

1.3.4 4. Security Standards Data

Preparation Scripts: - prepare_owasp_asvs.py: OWASP ASVS controls (17 sample controls) - prepare_nist.py: NIST Cybersecurity Framework (24 sample controls) - prepare_iso27001.py: ISO 27001:2022 Annex A (26 sample controls)

Total: 67 security controls prepared across 3 standards

1.3.5 5. Sample Inputs

Three ready-to-use examples: 1. Task Management App (inputs/requirements.txt) - Default 2. E-Commerce Platform (inputs/sample_ecommerce.txt) 3. Telemedicine Platform (inputs/sample_healthcare.txt)

1.3.6 6. Helper Scripts

  • setup.sh: Complete automated setup
  • run_example.sh: Easy execution of different examples
  • docker-compose.yml: Weaviate database setup

1.5 Usage

1.5.1 Quick Start

# 1. Setup (one-time)
./setup.sh

# 2. Run with default input
crewai run

# 3. Run with specific example
./run_example.sh ecommerce

1.5.2 Manual Steps

# 1. Configure environment
cp env.template .env
# Edit .env with OPENAI_API_KEY

# 2. Install dependencies
crewai install

# 3. Start Weaviate
docker-compose up -d

# 4. Prepare and ingest data
python -m security_requirements_system.data.prepare_owasp_asvs
python -m security_requirements_system.data.prepare_nist
python -m security_requirements_system.data.prepare_iso27001
python -m security_requirements_system.tools.weaviate_setup

# 5. Run the system
INPUT_FILE=inputs/sample_ecommerce.txt crewai run
